Слои

Содержание=>Слои

В системе реализована хорошо себя зарекомендовавшая система слоев. Т.е., каждый графический элемент располагается в одном из зарегестрированных слоев. Слой служит для группировки графических элементов. Слои различаются по идентификаторам. Идентификатор состоит из двух частей: класса и модификатора. Также идентификатор может быть любой последовательностью знаков. Наличие встроенных классов и модификаторов позволяют поддерживать согласованную систему слоев на различных рабочих местах вне зависимости от количества практически используемых слоев.

На локальной машине для каждого слоя (идентификатора) задаются:

  • Наименование. Любая последовательность букв и знаков.
  • Цвет. Любой из разрешенных в Вашей системе цветов. Если заданный Вами цвет не может отобразиться Вашей системой, то подбирается ближайший из разрешенных цветов.
  • Состояние. Слой может находиться в одном из следующих состояний:
    • слой выключен. Элементы нарисованные в таком слое на экране и на принтере не отображаются и для редактирования не доступны.
    • слой включен. Элементы нарисованные в таком слое отображаются на экране и на принтере, однако доступ к ним для их редактирования закрыт, т.е. редактирование элементов нарисованных во включенном слое невозможно.
    • слой редактируется. Элементы нарисованные в таком слое отображаются на экране и на принтере, кроме того их можно редактировать.
  • Тип трассировочных элементов на слое. Т.е. данный слой используется в процессе трассировки и на нем отображаются соответствующие элементы:
    • площадки - используется для отображения площадок
    • маска - используется для отображения паяльной маски
    • трафарет - используется для отображения отверстий в трафарете для трафаретных принтеров
    • отверстия - отверстия в плате
    • дорожки - используется для отображения трассировочных дорожек
    • полигоны - используется для отображения трассировочных полигонов
    • граница - используется для отображения границ трассировки
    • запрет - области запрета трассировки
  • Парный слой. Парный слой используется для отображения элементов с данного слоя при перемещении компонента на нижнюю сторону платы.

Система слоев образует стек слоев отображения. Помимо стека слоев отображения существует еще один стек слоев - стек трассировки. Отображение слоев на стек трассировки осуществляется автоматически с помощью задания модификатора в идентификаторе слоя. Несколько визуальных слоев могут отображаться на один слой трассировки. Например, слои Площадки сверху и Дорожки сверху отображаются на верхний трассировочный слой.

Существует определенная система слоев по умолчанию, однако допускается добавление, изменение и удаление слоев. При сохранении проекта слои сохраняются по идентификаторам. Текущее состояние слоев сохраняется на локальной машине. Поэтому, если Вы загружаете чужой проект, то он будет отображаться Вашими цветами, а не цветами разработчика проекта. Т.е. любой загруженный Вами проект будет иметь привычный Вам вид. Если проект содержит слои, которых нету в вашей системе, то они автоматически добавляются со значениями по умолчанию.

Добавление слоев может осуществляться и вручную.

Удаление слоев может осуществляться только вручную и только в том случае, если на удаляемых слоях во всех загруженных проектах нет никакой информации. Если Вы загрузите проект, в котором присутствует удаленный слой, то он автоматически добавится.
